# 📱 Museum Map — 유저 플로우 (User Flows)

> 모든 기능의 사용자 행동 흐름을 정리한 문서입니다.
> 마지막 업데이트: **2026-03-21**

---

## 1. 핵심 루프 (Core Loop)

```
탐색 → 저장 → 동선 생성 → 방문 → 리뷰 → 컬렉션 공유 → 재유입
```

---

## 2. 지도 탐색 (Map Exploration)

**경로**: `/` (메인)

```
앱 진입
  → 첫 방문: 로딩 오버레이 (3,379개 미술관 로딩)
  → 재방문: sessionStorage 캐시 → 즉시 표시
  → 지도에서 미술관 마커 탐색
  → 마커 클릭 → 미술관 카드 표시 (하단 슬라이드)
  → 카드에서 주요 정보 확인 (이름, 유형, 위치)
```

**필터링**:
```
전체 / 미술관 / 박물관 / 갤러리 필터 칩 선택
  → 지도 마커 필터링
  → 카드 리스트 필터링
```

---

## 3. 검색 (Search)

**경로**: `/` (메인 상단 검색창)

```
검색창 클릭 (회색 아웃라인 → 보라색 아웃라인 전환)
  → 키워드 입력
  → 실시간 드롭다운 (최대 8개 결과)
  → 한국어/영어/도시/국가 이름으로 검색 가능
  → 결과 클릭 → 해당 미술관으로 지도 이동 + 카드 표시
```

**검색창 스타일**: 모바일/태블릿/PC 동일 (border-gray-300 → border-purple-500 on focus)

**다국어 검색**:
- 한국어 locale → 한국어 이름 우선 표시 (영어 부명칭)
- 영어 locale → 영어 이름 우선 표시 (한국어 부명칭)

---

## 3-1. AI 추천 검색

**경로**: `/` (메인 상단 검색창)

```
자연어 입력 (e.g. "인상파 그림 볼 수 있는 곳")
  → Gemini 2.5 Flash이 의도 파싱
  → DB에서 관련 미술관 검색
  → AI가 각 미술관별 추천 이유 생성 (로칼 언어)
  → 결과 카드에 카테고리 배지 + 추천 이유 표시
  → 카드 클릭 → 미술관 상세로 이동
```

---

## 4. 미술관 상세 (Museum Detail)

**경로**: `/museums/[id]` (SSR)

```
> 미술관 카드 클릭 또는 직접 URL 접근
>   → 서버에서 데이터 프리로드 (SSR)
>   → 기본 정보 표시 (이름, 설명, 위치, 유형)
>   → 전시 정보 (Serper API, 7일 캐시)
>   → 관련 작품 목록
>   → 리뷰 목록
>   → CTA: 저장 / 방문 체크 / 리뷰 작성 / 동선 추가
>   → 공유 버튼 → UTM 파라미터 자동 부착 (utm_source=share&utm_medium=social&utm_campaign=museum_share)
```

---

## 5. 미술관 저장 (Save)

**경로**: `/` 또는 `/museums/[id]` → API `POST /api/collections/save`

```
미술관 카드에서 "저장하기" 버튼 클릭
  → 로그인 확인 (미로그인 시 로그인 유도)
  → 기본 폴더 "내 저장소"에 저장
  → 또는 특정 폴더 선택
  → 토스트 알림: "저장되었습니다"
  → 저장 목록에서 확인 가능 (/saved)
```

---

## 6. 여행 동선 (Plan / AutoRoute)

**경로**: `/plans/new`

```
"동선 만들기" 클릭
  → 저장된 미술관 목록에서 선택
  → "자동 동선 생성" 클릭
  → 선택한 미술관들의 위치 기반 최적 순서 계산
  → 지도 위에 동선 시각화
  → Plan 이름 입력 → 저장
  → 생성된 동선은 /plans에서 확인
```

**동선 상세 (`/plans/[id]`)**:
```
동선 이름 + 미술관 순서 목록
  → 각 미술관 카드 → 방문 체크 가능
  → 미술관 순서 변경 가능
```

---

## 7. 컬렉션 공유 (Collection)

**경로**: `/collections`

```
내 컬렉션 목록 확인
  → 새 컬렉션 생성 (이름, 설명)
  → 저장한 미술관 + 방문 리뷰 추가
  → "Public" 전환 → 공유 URL 생성 (UTM 자동 부착)
  → URL 복사 또는 SNS 공유
  → 다른 사용자가 URL 접속 → 컬렉션 뷰 확인
  → "내 지도로 가져오기" → 1번 탐색으로 재유입
```

---

## 8. MM 스토리 (Blog)

**경로**: `/blog` → `/blog/[id]`

```
홈 피드 또는 하단 네비에서 스토리 탭 클릭
  → 스토리 카드 리스트 (가로 스크롤)
  → 스토리 카드 클릭 → 상세 (SSR)
  → 미술관/작품 소개 콘텐츠 읽기
  → 관련 미술관 링크 클릭 → 지도로 이동
  → 관련 작품 링크 클릭 → 작품 상세로 이동
```

**어드민 등록 플로우**:
```
/admin에서 스토리 작성 (한국어)
  → 제목, 본문, 이미지, 관련 미술관/작품 선택
  → 발행 → 자동 번역 트리거 (첫 조회 시)
```

---

## 9. 작품 상세 (Artwork Detail)

**경로**: `/artworks/[id]` (SSR)

```
스토리 또는 미술관 상세에서 작품 카드 클릭
  → 서버에서 데이터 프리로드 (SSR, initialData)
  → 작품 이미지 + 제목 + 작가 + 설명 표시
  → 소장 미술관 정보
  → 관련 스토리 링크
```

### 9-1. 작품 목록 (/artworks)

```
작품 탭 클릭 → 903개 작품 목록 표시
  → 실시간 검색 (제목/작가/미술관명, 300ms debounce)
  → 작품 카드 클릭 → 상세 이동
  → 뒤로가기 → 이전 스크롤 위치 유지 (sessionStorage)
```

---

## 9-2. 사진 업로드 (Photo Upload)

**경로**: `/museums/[id]` → 사진 업로드 버튼

```
미술관 상세에서 "사진 업로드" 클릭
  → 로그인 확인 (미로그인 시 로그인 유도)
  → 사진 선택 (JPG/PNG/WebP, 최대 5MB)
  → 일일 제한 확인 (최대 10장/일)
  → 업로드 → 관리자 승인 대기 상태
  → 승인 후 서비스 전반에 표시
```

## 11. 다국어 (i18n)

**사용자 경험**:
```
언어 설정 변경 (설정 > 언어)
  → locale 변경 → UI 재렌더링
  → 한국어: DB 필드 직접 (nameKo, titleKo)
  → 영어: DB 필드 직접 (nameEn, titleEn)
  → 기타 11개 언어: TranslationCache 조회
    → 캐시 없으면 Gemini 자동 번역 → DB 저장
    → 이후 즉시 표시
```

**고유명사 처리**:
```
미술관/작품 고유명사
  → 한국어: "국립현대미술관 과천관"
  → 일본어: "国立現代美術館 果川館"  
  → 영어 약어: "MoMA" (모든 언어에서 유지)
```

## 12. 인증 (Authentication)

**경로**: `/login`

```
로그인 화면 진입 (디졸브 페이드인 0.8s)
  → NavHeader 표시 (로고 클릭 → 홈, 브라우저 뒤로가기 가능)
  → 배경 이미지 좌→우 루프 패닝 (50초 주기)
  → 비맵 페이지이므로 스플래시 스킵 (pathname !== '/')
  → "Google로 계속하기" 클릭
  → 선택 시트 (모바일: 바텀시트 / PC: 팝업)
    → "로그인" 또는 "회원가입" 선택
    → 회원가입 시: 동의 모달 표시
      → 이용약관 / 개인정보처리방침 체크
      → 각 항목 클릭 → 오른쪽 슬라이드 패널로 상세 확인 (fadeSlideIn 애니메이션)
      → 백버튼 → 동의 화면으로 복귀
      → "전체 보기" → router.push()로 /terms 즉시 이동 (클라이언트 사이드)
      → 수집 정보: 이메일, 프로필 이미지 (이름 수집 안 함)
      → 전체 동의 후 → Google OAuth 진행
  → 로그인 완료 → consentChecking 상태로 맵 플래시 방지 → 이전 페이지로 복귀
  → 세션 유지 (NextAuth.js)
```

## 13. 피드백 (Feedback)

**경로**: `/feedback`

```
설정 또는 하단 피드백 링크 클릭
  → 피드백 유형 선택 (버그, 제안, 기타)
  → 내용 입력 → 제출
```

## 상태 유지 전략

| 페이지 간 이동 시 | 방법 |
|-----------------|------|
| 지도 줌/위치 | URL 쿼리 파라미터 (`?lat=&lng=&zoom=`) |
| 미술관 데이터 | sessionStorage 캐시 (첫 로딩 후 유지) |
| 번역 캐시 | localStorage + DB TranslationCache |
| 인증 상태 | NextAuth.js 세션 |
| 작품 목록 스크롤 | sessionStorage (artworks-scroll-pos) |

## 14. 어드민 (Admin)

**경로**: `/admin`

```
관리자 인증 → 대시보드
  → 요약 카드 클릭 → 해당 탭으로 이동
    - 가입 사용자 → users 탭
    - 작성된 스토리 → blog 탭
    - 등록된 미술관/박물관 → museums 탭
  → GA4/AI 메트릭 카드 클릭 → analytics/ai 탭
  → API 사용량 카드 클릭 → ai 탭
```
